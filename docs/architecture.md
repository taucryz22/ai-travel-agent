# Архитектура проекта

## Общая идея

Проект состоит из двух основных частей:

- **frontend** — пользовательский интерфейс
- **backend** — API и логика построения маршрута

Система построена так, чтобы:
1. понять текстовый запрос пользователя,
2. определить город и интересы,
3. найти реальные места,
4. рассчитать маршрут между ними,
5. вернуть готовый план поездки.

---

## Высокоуровневая схема

```text
Пользователь
   ↓
Frontend (React)
   ↓
Backend (FastAPI)
   ↓
1. Разбор запроса (LLM + fallback)
2. Поиск города / валидация через OSM
3. Подбор мест через OSM
4. Получение текстовых подсказок из Wikivoyage
5. Планировщик маршрута
6. Расчёт перемещений через ORS
   ↓
Готовый маршрут
